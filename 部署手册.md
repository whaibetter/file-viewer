# File Viewer 部署手册

## 环境要求

- Ubuntu/Debian 系统
- Python 3.8+
- Nginx（推荐用于反向代理）
- systemd

## 快速部署

```bash
# 1. 克隆或上传项目到服务器
git clone <repo-url> /home/file-viewer
cd /home/file-viewer

# 2. 运行安装脚本
sudo ./install.sh

# 3. 修改默认密码
sudo file-viewer passwd
```

安装完成后访问 `http://<服务器IP>` 即可使用。

## 开发模式（热重载）

服务直接运行项目目录中的代码，修改前后端代码后只需重启即可：

```bash
# 修改 index.html 或 file-viewer-server.py 后
sudo file-viewer restart
```

无需复制文件，重启即生效。

## 目录结构

```
/home/file-viewer/           # 项目目录（服务直接读取）
├── config.yaml              # 配置文件
├── file-viewer              # 管理脚本
├── file-viewer-server.py    # 后端服务
├── file-viewer.service      # systemd 服务配置
├── index.html               # 前端页面
├── install.sh               # 安装脚本
├── README.md                # 说明文档
└── 部署手册.md              # 本文档

/etc/file-viewer/            # 系统数据目录
├── config.yaml              # 用户配置（可选）
└── passwd                   # 密码文件
```

## 管理命令

```bash
file-viewer start      # 启动服务
file-viewer stop       # 停止服务
file-viewer restart    # 重启服务（加载最新代码）
file-viewer status     # 查看状态
file-viewer passwd     # 修改密码
file-viewer logs       # 查看日志
file-viewer logs -f    # 实时查看日志
```

## Nginx 配置（推荐）

后端服务运行在 `127.0.0.1:9001`，需要 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:9001;
        proxy_set_header Host $host;
        proxy_read_timeout 30s;
    }

    # 前端页面
    location / {
        proxy_pass http://127.0.0.1:9001;
        proxy_set_header Host $host;
    }
}
```

## 配置文件位置

| 配置项 | 配置路径 | 说明 |
|--------|----------|------|
| 主配置 | `/home/file-viewer/config.yaml` | 项目目录 |
| 用户配置 | `/etc/file-viewer/config.yaml` | 覆盖主配置（可选） |
| 密码文件 | `/etc/file-viewer/passwd` | SHA256哈希存储 |

## 常见问题

### 服务无法启动

```bash
# 查看错误日志
sudo journalctl -u file-viewer -n 50

# 检查端口是否被占用
sudo lsof -i :9001
```

### 修改代码后不生效

```bash
sudo file-viewer restart
```

### 无法删除文件

检查该路径是否在删除白名单中，通过设置面板添加。

### 更新项目

```bash
cd /home/file-viewer
git pull
sudo file-viewer restart
```
